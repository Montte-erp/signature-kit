[Licitei](https://licitei.com.br) builds its documents with [`@react-pdf/renderer`](https://react-pdf.org) and marks where the signature belongs with its own React component. That is a great authoring story and a bad signing story: the bytes that come out of `@react-pdf/renderer` are a plain PDF with no signature field, and the A1 certificate that has to sign them lives in object storage behind a presigned URL — not on the server's disk.

So how do you turn a beautiful-but-unsigned react-pdf document into a PAdES-signed PDF, with a rubric on every page, using a certificate you never want to write to disk? That is exactly the seam `@signature-kit` was built for — and it closes the gap without caring how the PDF was produced.

This post walks the flow end to end. We render the template, stamp a visible rubric on every page, resolve where the visible signature goes (the marked field if Licitei placed one, otherwise auto-placement), and provide the `Signatures` boundary straight from the presigned `.pfx` URL.

```package-install
@signature-kit/react @signature-kit/pdf @signature-kit/a1 @react-pdf/renderer
```

<Callout type="info" title="One flow, two placement modes">
  If the React-PDF template carries a signature field, the visible signature lands exactly there
  (`pdfSignatureAppearanceFromField`). If it does not, `signPdf` places it for you with
  `appearance.placement.kind: "auto"`. Same signer, same certificate, same PAdES output.
</Callout>

## The flow

<Steps>
  <Step title="Render the React-PDF template to bytes">
    `renderReactSignatureTemplatePdf` validates the template and the render options and produces the draft PDF bytes
    as an `Effect`. This is the document Licitei assembled — pages, labels, and the marked signature position — turned
    into a `Uint8Array` ready to stamp and sign.

    ```ts title="1-render.ts"
    import { renderReactSignatureTemplatePdf } from "@signature-kit/react/react-pdf"
    import { Effect } from "effect"

    // `template` is your ReactSignatureTemplate — the same object Licitei marks
    // the signature position on with its custom field component.
    const pdfBytes: Uint8Array = await Effect.runPromise(
      renderReactSignatureTemplatePdf(template, { title: "Edital — proposta" }),
    )
    ```

    ```ts
    renderReactSignatureTemplatePdf(
      template: ReactSignatureTemplate,
      options?: ReactPdfRenderOptions,
    ): Effect.Effect<Uint8Array, ReactIntegrationError>
    ```
  </Step>

  <Step title="Stamp a name-derived rubric on every page">
    A PAdES signature is a single CMS over the whole document; its widget carries no visible appearance. The Brazilian
    *rubrica em todas as páginas* is therefore drawn as ordinary page content **before** signing — one
    `stampPdfRubric` call with `pages: "all"`, so the same rubric repeats on every page and the later signature's byte
    range covers all of it.

    `rect` is `[left, bottom, right, top]` in PDF points with a **bottom-left origin**. Keeping `left` small pins the
    rubric to the left margin of each page.

    ```ts title="2-stamp.ts"
    import { stampPdfRubric } from "@signature-kit/pdf/stamp"
    import { Effect } from "effect"

    // Name-derived initials: "Maria Auxiliadora Costa" -> "MAC".
    const initials = (name: string) =>
      name
        .split(/\s+/)
        .filter(Boolean)
        .map((part) => part[0]!.toUpperCase())
        .join("")

    const stampedPdf: Uint8Array = await Effect.runPromise(
      stampPdfRubric(pdfBytes, {
        rect: [24, 24, 132, 78], // left margin, bottom-left origin
        pages: "all",
        lines: [initials("Maria Auxiliadora Costa")],
        border: true,
      }),
    )
    ```

    ```ts
    stampPdfRubric(
      pdf: Uint8Array,
      stamp: PdfRubricStamp,
    ): Effect.Effect<Uint8Array, PdfError>
    ```

    <TypeTable
      type={{
        rect: { type: "[number, number, number, number]", required: true, description: "[left, bottom, right, top] in PDF points, bottom-left origin." },
        pages: { type: '"all" | number[]', description: 'Which pages to stamp. Default: "all".' },
        lines: { type: "string[]", description: "Text lines to draw — e.g. the name-derived initials." },
        imagePng: { type: "Uint8Array", description: "PNG bytes to draw instead of (or alongside) the lines." },
        border: { type: "boolean", description: "Draw the framing rectangle. Default: true." },
      }}
    />
  </Step>

  <Step title="Resolve where the visible signature goes">
    If Licitei marked a signature field on the template, derive the appearance from it — the visible signature lands on
    that exact rect. `pdfSignatureAppearanceFromField` converts the field's top-left builder rect into the bottom-left
    `widgetRect` the PAdES signer expects.

    ```ts title="3a-marked-field.ts"
    import { pdfSignatureAppearanceFromField } from "@signature-kit/react/builder"
    import type { PdfSignatureAppearance } from "@signature-kit/pdf/config"
    import { Effect } from "effect"

    const appearance: PdfSignatureAppearance = await Effect.runPromise(
      pdfSignatureAppearanceFromField(template, "field-signer-1"),
    )
    // -> { pageIndex, widgetRect: [left, bottom, right, top] }
    ```

    If there is no marked field, hand the placement to `signPdf` with `kind: "auto"`. It looks for an existing empty
    signature slot that matches the anchor and, finding none, picks a collision-free rect on the chosen page.

    ```ts title="3b-auto-placement.ts"
    import type { PdfSignatureAppearance } from "@signature-kit/pdf/config"

    const appearance: PdfSignatureAppearance = {
      placement: {
        kind: "auto",
        page: "last",          // "first" | "last"; default "last"
        anchor: "bottom-right",
        width: 180,
        height: 54,
        margin: 36,
        gap: 8,
      },
    }
    ```

    <Callout type="warn" title="Auto-placement is only reachable through signPdf">
      The browser helper `signBrowserPdf` always derives `{ pageIndex, widgetRect }` from a builder field and never sets
      `placement`. To use `kind: "auto"` you must call `signPdf` directly with an `appearance` object — which is exactly
      what this flow does.
    </Callout>
  </Step>

  <Step title="Provide the A1 certificate from a presigned URL and sign">
    `a1SignaturesLayerFromUrl` fetches the PKCS#12 (`.pfx`) bytes from the presigned URL with a `GET`, decrypts them
    with the password, and exposes the agnostic `Signatures` service as a `Layer`. `signPdf` requires that boundary, so
    you `Effect.provide` the layer before running. The certificate is streamed straight into memory — it never touches
    the server's disk.

    The password is a `Redacted<string>`, so it stays out of logs and error messages. Wrap it with `Redacted.make`.

    ```ts title="4-sign.ts"
    import { a1SignaturesLayerFromUrl } from "@signature-kit/a1/signer"
    import { signPdf } from "@signature-kit/pdf/sign"
    import { Effect, Redacted } from "effect"

    const signaturesLayer = a1SignaturesLayerFromUrl({
      url: presignedPfxUrl,                 // e.g. an S3 GET URL with a token
      password: Redacted.make(pfxPassword),
      // headers: { Authorization: `Bearer ${token}` }, // if not baked into the URL
    })

    const signedPdf: Uint8Array = await Effect.runPromise(
      signPdf({
        pdf: stampedPdf,
        appearance,                 // marked field OR auto placement, from step 3
        reason: "Assinatura da proposta",
        name: "Maria Auxiliadora Costa",
        policy: "pades-ades",       // local PAdES — no network policy fetch
      }).pipe(Effect.provide(signaturesLayer)),
    )
    ```

    ```ts
    a1SignaturesLayerFromUrl(
      source: A1RemoteSource,
    ): Layer.Layer<Signatures, SignatureKitError>

    signPdf(
      input: PdfSigningRequest,
    ): Effect.Effect<Uint8Array, PdfError | CmsError | SignatureKitError, Signatures>
    ```

    <Callout type="info" title="Need the raw bytes first?">
      To inspect or reuse the container before signing, `fetchA1Pkcs12({ url, headers? })` returns the `.pfx` bytes as
      `Effect.Effect<Uint8Array, SignatureKitError>`. `a1SignaturesLayerFromUrl` is built on top of it.
    </Callout>
  </Step>
</Steps>

## End to end

The four steps as one Effect program. The marked field is preferred; auto-placement is the fallback when the template
carries no signature field.

```ts title="sign-licitei-document.ts"
import { renderReactSignatureTemplatePdf } from "@signature-kit/react/react-pdf"
import { pdfSignatureAppearanceFromField } from "@signature-kit/react/builder"
import { stampPdfRubric } from "@signature-kit/pdf/stamp"
import { signPdf } from "@signature-kit/pdf/sign"
import { a1SignaturesLayerFromUrl } from "@signature-kit/a1/signer"
import type { ReactSignatureTemplate } from "@signature-kit/react/config"
import type { PdfSignatureAppearance } from "@signature-kit/pdf/config"
import { Effect, Redacted } from "effect"

type SignInput = {
  readonly template: ReactSignatureTemplate
  readonly signerName: string
  readonly markedFieldId?: string // set by Licitei's signature component, if any
  readonly presignedPfxUrl: string
  readonly pfxPassword: string
}

const initials = (name: string) =>
  name
    .split(/\s+/)
    .filter(Boolean)
    .map((part) => part[0]!.toUpperCase())
    .join("")

const signLiciteiDocument = (input: SignInput) =>
  Effect.gen(function* () {
    // 1. Render the React-PDF template Licitei assembled.
    const pdfBytes = yield* renderReactSignatureTemplatePdf(input.template, {
      title: "Edital — proposta",
    })

    // 2. Rubric on the left of every page (covered by the signature's byte range).
    const stamped = yield* stampPdfRubric(pdfBytes, {
      rect: [24, 24, 132, 78],
      pages: "all",
      lines: [initials(input.signerName)],
      border: true,
    })

    // 3. Marked field when present, else auto-placement.
    const appearance: PdfSignatureAppearance =
      input.markedFieldId === undefined
        ? { placement: { kind: "auto", page: "last", anchor: "bottom-right", width: 180, height: 54 } }
        : yield* pdfSignatureAppearanceFromField(input.template, input.markedFieldId)

    // 4. Provide the A1 boundary from the presigned URL and sign.
    return yield* signPdf({
      pdf: stamped,
      appearance,
      reason: "Assinatura da proposta",
      name: input.signerName,
      policy: "pades-ades",
    }).pipe(
      Effect.provide(
        a1SignaturesLayerFromUrl({
          url: input.presignedPfxUrl,
          password: Redacted.make(input.pfxPassword),
        }),
      ),
    )
  })

const signedPdf: Uint8Array = await Effect.runPromise(
  signLiciteiDocument({
    template,
    signerName: "Maria Auxiliadora Costa",
    markedFieldId: "field-signer-1", // omit for auto-placement
    presignedPfxUrl: "https://storage.example.com/cert.pfx?token=…",
    pfxPassword: "…",
  }),
)
```

## Why this works

Step back and the four moving parts each solve a distinct piece of the Licitei problem:

- **Agnostic to how the PDF was built.** `signPdf` takes bytes. It does not know or care that `@react-pdf/renderer`
  produced them — the same signer works for templates, scanned documents, or anything else. Licitei keeps its authoring
  stack untouched and gains PAdES signing.
- **The marked position is honoured, with a safety net.** When Licitei's component drops a signature field,
  `pdfSignatureAppearanceFromField` lands the visible signature on that exact rect. When it does not, `kind: "auto"`
  finds a clean spot instead of failing. One code path covers both documents.
- **A1 by presigned URL — the certificate never hits disk.** `a1SignaturesLayerFromUrl` streams the `.pfx` from object
  storage straight into the `Signatures` boundary in memory. No temp files, no certificate checked into the server, and
  the password stays `Redacted`. Rotating the certificate is just rotating the URL.
- **One signature, rubric on every page.** Stamping before signing means a single CMS backs the rubric repeated across
  all pages — not one signature per page — which is what the *rubrica em todas as páginas* requirement actually wants.

<Callout type="warn" title="Keep it local: pades-ades">
  This flow signs with `policy: "pades-ades"`, which is fully local. `pades-icp-brasil` fetches the ICP-Brasil policy
  over the network — avoid it unless you specifically need the ICP-Brasil signature policy embedded.
</Callout>
