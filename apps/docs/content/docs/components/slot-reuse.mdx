title: Slot reuse
description: "Best-guess placement reuses a signature slot the document already carries — your app never sends coordinates. SignatureKit detects an empty /Sig widget and drops the signature into it."
---

Best-guess placement reuses a slot the document already carries — no coordinates from your app. When a PDF arrives with an **empty signature widget** (`/Subtype /Widget`, `/FT /Sig`, no `/V`), automatic placement reuses that slot instead of inventing a rectangle. If no slot exists, it falls back to anchor geometry.

```package-install
@signature-kit/pdf @signature-kit/a1
```

<Callout type="info" title="This is how a marked position travels">
  A "pre-marked" PDF is just a normal PDF whose author dropped an empty `/Sig` widget where the
  signature should go. Licitei's custom component assembles the document with that empty slot;
  SignatureKit detects it and fills it. The position is carried **in the PDF**, not in an API call.
</Callout>

## How an empty slot is detected

Auto placement scans the page annotations for a signature widget that has no value yet. The predicate is exact — a `/Widget` whose field type is `/Sig` with no `/V`:

```ts title="placement.ts (pageSignatureSlots)"
const subtype = annotation?.lookupMaybe(PDFName.of("Subtype"), PDFName)?.asString();
const fieldType = annotation?.lookupMaybe(PDFName.of("FT"), PDFName)?.asString();
const rect = rectArray === undefined ? undefined : rectangleFromArray(rectArray);
if (
  object !== undefined &&
  subtype === "/Widget" &&
  fieldType === "/Sig" &&
  annotation?.get(PDFName.of("V")) === undefined &&
  rect !== undefined
) {
  slots.push({ object, rect });
}
```

A widget that already carries a `/V` is a *signed* field and is skipped — reuse only targets empty slots.

## Picking among several slots

A document can carry more than one empty slot. The anchor decides which one wins: each slot is scored by the squared distance between its anchor point and the page-bounds anchor point, and the nearest slot is chosen.

```ts title="placement.ts (chooseExistingSignatureSlot)"
const chooseExistingSignatureSlot = (
  bounds: PdfRectangle,
  slots: ReadonlyArray<ExistingSignatureSlot>,
  anchor: PdfSignatureAnchor,
): ExistingSignatureSlot | undefined => {
  let bestSlot: ExistingSignatureSlot | undefined;
  let bestScore = Number.POSITIVE_INFINITY;

  for (const slot of slots) {
    const score = squaredAnchorDistance(slot.rect, bounds, anchor);
    if (score < bestScore) {
      bestScore = score;
      bestSlot = slot;
    }
  }

  return bestSlot;
};
```

There are nine anchors — `bottom-left`, `bottom-center`, `bottom-right`, `middle-left`, `middle-center`, `middle-right`, `top-left`, `top-center`, `top-right`. The default is `bottom-right`. In the demo above, `top-left` selects `Declarante`, `middle-center` selects `Testemunha`, and `bottom-right` (the default, used by **Best guess**) selects `Responsável legal`.

## Signing into a slot

Slot reuse lives in `resolveAutoPlacement`, which is reached **only** through `appearance.placement.kind === "auto"`. Drop to the PDF-layer `signPdf` and pass an auto placement — omit the anchor to use the `bottom-right` default:

```ts title="sign.ts"
import { signPdf } from "@signature-kit/pdf/sign"
import { a1SignaturesLayer } from "@signature-kit/a1/signer"
import { Effect, Redacted } from "effect"

const signed = signPdf({
  pdf: bytes,
  reason: "Signed into a pre-marked slot",
  name: "Maria de Souza",
  signatureLength: 16384,
  policy: "pades-ades",
  appearance: {
    placement: { kind: "auto", pageIndex: 0, anchor: "bottom-right" },
  },
}).pipe(
  Effect.provide(a1SignaturesLayer({ pfx, password: Redacted.make(password) })),
)
```

The empty widget the library picks is filled with the signature's `/V`. Reading the page back, exactly one widget now carries a `/V` — at the slot's original rect.

<Callout type="warn" title="signBrowserPdf can't reuse a slot">
  `signBrowserPdf` derives a **manual** widget rect from the template field, so it never triggers
  reuse. To reuse an existing `/Sig` slot, call `signPdf` with `appearance.placement.kind: "auto"` —
  that is the only path through `resolveAutoPlacement`.
</Callout>

## Geometry fallback

If the page has no empty `/Sig` slot, auto placement falls back to synthesising a rectangle: it lays out candidate positions inside the page margins, drops any that collide with existing annotations, and keeps the one nearest the anchor.

```ts title="placement.ts (resolveAutoPlacement)"
const existingSlot = chooseExistingSignatureSlot(
  bounds,
  pageSignatureSlots(pdfDoc, annotations),
  anchor,
);
if (existingSlot !== undefined) {
  return Effect.succeed({
    pageIndex,
    widgetRect: tupleFromRectangle(existingSlot.rect),
    existingWidgetObject: existingSlot.object,
  });
}
// …no slot: search the margins for a non-colliding rectangle
const obstacles = pageAnnotationRectangles(pdfDoc, annotations);
const chosen = chooseAutoRect(bounds, obstacles, width, height, gap, anchor);
```

So the same call works for both shapes of document: reuse a marked slot when one exists, otherwise pick a sensible empty spot.

## Marking a slot from your own app

To pre-mark a position, assemble the PDF with an empty `/Sig` widget — the field type is `/Sig`, the subtype is `/Widget`, and there is no `/V`. This is the exact shape the demo generates:

```ts title="mark-slot.ts"
import { PDFArray, PDFDocument, PDFName, PDFNumber, PDFString } from "@cantoo/pdf-lib"

const pdf = await PDFDocument.create()
const page = pdf.addPage([320, 180])

const annotations = pdf.context.obj([])
const fields = pdf.context.obj([])
for (const slot of [
  { label: "Declarante", rect: [20, 118, 140, 158] },
  { label: "Testemunha", rect: [100, 68, 220, 108] },
  { label: "Responsável legal", rect: [180, 20, 300, 60] },
]) {
  const rect = PDFArray.withContext(pdf.context)
  for (const n of slot.rect) rect.push(PDFNumber.of(n))
  const widget = pdf.context.obj({
    Type: "Annot",
    Subtype: "Widget",
    FT: "Sig",
    Rect: rect,
    T: PDFString.of(slot.label),
    F: 4,
    P: page.ref,
  })
  const widgetRef = pdf.context.register(widget)
  annotations.push(widgetRef)
  fields.push(widgetRef)
}
page.node.set(PDFName.of("Annots"), annotations)
pdf.catalog.set(PDFName.of("AcroForm"), pdf.context.register(pdf.context.obj({ Fields: fields })))

const bytes = await pdf.save({ useObjectStreams: false, updateFieldAppearances: false })
```

Hand those bytes to `signPdf` with an auto placement and SignatureKit drops the signature into the slot.
