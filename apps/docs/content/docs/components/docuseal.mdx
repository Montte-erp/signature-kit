---
title: DocuSeal Bridge
description: "Build the token payload for DocuSeal's embedded builder from a typed input — the HS256 JWT and the secret stay on the backend."
---

`createDocuSealBuilderTokenPayload` translates a typed input (`DocuSealBuilderInput`) into the _snake_case_ payload that DocuSeal's embedded builder expects inside its session token. It validates the input, ensures you passed a template **or** at least one document, and returns an object ready to become the `payload` of a JWT — without ever touching the signing secret.

```ts title="docuseal-token.ts"
import { createDocuSealBuilderTokenPayload } from "@signature-kit/react/builder"
import { Effect } from "effect"

const payload = await Effect.runPromise(
  createDocuSealBuilderTokenPayload({
    userEmail: "signer@example.com",
    templateId: 1000001,
    externalId: "contract-42",
    extractFields: true,
  }),
)
// → { user_email: "signer@example.com", template_id: 1000001, external_id: "contract-42", extract_fields: true }
```

<Callout type="error" title="The secret never crosses into the browser">
  `createDocuSealBuilderTokenPayload` only builds the **payload**. Signing the JWT (HS256, with the `DOCUSEAL_API_KEY`) is
  your backend's responsibility. The `@signature-kit/react` package never serializes or embeds the secret into the browser
  bundle — generate the token on the server and send only the already-signed JWT to the client.
</Callout>

## Input

The function decodes the argument against `DocuSealBuilderInputSchema`. Missing optional fields simply do not appear in the final payload.

<TypeTable
  type={{
    userEmail: { type: "string", required: true, description: "Email of the DocuSeal user owning the builder session. Becomes user_email." },
    integrationEmail: { type: "string", description: "Integration email to assign the submission to. Becomes integration_email." },
    templateId: { type: "number", description: "Id of an existing DocuSeal template to open in the builder. Becomes template_id." },
    externalId: { type: "string", description: "Your external reference to correlate the submission. Becomes external_id." },
    folderName: { type: "string", description: "DocuSeal folder where the document will be created. Becomes folder_name." },
    documentUrls: { type: "readonly string[]", description: "URLs of documents to load when there is no templateId. Becomes document_urls." },
    name: { type: "string", description: "Name of the created template/document. Becomes name." },
    extractFields: { type: "boolean", description: "Asks DocuSeal to automatically extract fields from the documents. Becomes extract_fields." },
  }}
/>

<Callout type="info" title="templateId or documentUrls">
  The payload needs at least one source path: either you pass `templateId`, or a non-empty `documentUrls`. With neither
  of them, the `Effect` fails with `ReactIntegrationError` (code `invalidBuilderInput`,
  operation `react.docuseal.payload`).
</Callout>

## Output payload

Success is a `DocuSealBuilderTokenPayload` with the _snake_case_ keys required by the DocuSeal builder. Only `user_email` is always present; the rest appear when the corresponding input field was provided.

## Signing the token on the backend

Build the payload, add the JWT claims, and sign with HS256 on the server. Only the resulting token goes to the client, which passes it to DocuSeal's embedded builder component.

```ts title="api/docuseal-token.ts"
import { createDocuSealBuilderTokenPayload } from "@signature-kit/react/builder"
import { Effect } from "effect"
import jwt from "jsonwebtoken"

export async function issueBuilderToken() {
  const payload = await Effect.runPromise(
    createDocuSealBuilderTokenPayload({
      userEmail: "signer@example.com",
      name: "Service Agreement",
      documentUrls: ["https://files.example.com/contract.pdf"],
      extractFields: true,
    }),
  )

  // HS256 + secret: backend only, never in the browser bundle.
  return jwt.sign(payload, process.env.DOCUSEAL_API_KEY!, { algorithm: "HS256" })
}
```
