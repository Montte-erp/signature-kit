---
title: SignatureBuilderSurface
description: "O construtor de assinaturas composto do @signature-kit/react: posicione campos sobre as páginas de um documento, com partes estilizáveis via data-slot."
---

`SignatureBuilderSurface` renderiza o construtor inteiro a partir de um `SignatureBuilderStore`: papéis, documentos, páginas e os campos posicionáveis. Ele desenha a _intenção_ de assinatura — quem assina o quê e onde — e nunca toca na chave. A assinatura em si permanece na camada de assinatura do `Signatures`.

```package-install
@signature-kit/react @signature-kit/pdf @signature-kit/a1
```

## Anatomia

A surface é composta: você pode renderizá-la inteira ou montar as partes manualmente para controlar o layout. Cada parte expõe um `data-slot`, então você estiliza por classe sem trocar a lógica.

```tsx title="ContractBuilder.tsx"
import { Effect } from "effect"
import {
  SignatureBuilderSurface,
  useSignatureBuilderStore,
} from "@signature-kit/react/components"

function ContractBuilder({ initialState }) {
  const store = useSignatureBuilderStore(initialState)

  return (
    <SignatureBuilderSurface
      store={store}
      onFieldPlacement={(placement) => void Effect.runPromise(store.placeField(placement))}
    />
  )
}
```

| Slot | Elemento | Descrição |
| ---- | ------- | ----------- |
| `signature-builder-surface` | div | A raiz da surface; envolve papéis e documentos. |
| `signature-builder-roles` | div | Lista dos papéis (signatários) do template. |
| `signature-builder-role` | button | Um papel selecionável. |
| `signature-builder-document` | div | Um documento do template. |
| `signature-builder-document-header` | div | Cabeçalho com o nome do documento. |
| `signature-builder-page` | div | Uma página renderizada do documento. |
| `signature-builder-page-label` | span | Rótulo da página (índice / nome). |
| `signature-builder-field-frame` | div | Frame clicável onde um campo é posicionado. |
| `signature-builder-field` | button | Um campo de assinatura já posicionado (clicável). |

<Callout type="info" title="A camada é sua">
  A surface não executa `Effect` por trás dos panos. Cada interação retorna um `Effect` (por exemplo, `store.placeField(...)`); a aplicação decide
  quando executá-lo com `Effect.runPromise` — normalmente no handler de evento do React. É ali que os erros viram UX.
</Callout>

## Partes compostas

Quando você precisa do seu próprio layout, construa-o a partir das subpartes em vez da surface inteira. Todas elas recebem o mesmo `store`:

- `SignatureBuilderRoles` — apenas a faixa de papéis.
- `SignatureBuilderDocument` — um único documento (`documentId`), com seu cabeçalho.
- `SignatureBuilderPage` — uma única página (`documentId`, `pageIndex`) com seus frames de campo.
- `SignatureBuilderField` — um campo posicionado individual.

<Callout type="info" title="Estado e seleção">
  Leia fatias do builder com [os hooks](/docs/components/hooks) (`useSignatureBuilderSelector`, `signatureBuilderSelectors`)
  e manipule campos com a [Builder API](/docs/components/builder-api). A surface é apenas a camada visual sobre esse store.
</Callout>
