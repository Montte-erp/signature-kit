`@signature-kit/react` agora é uma camada fina de navegador/React sobre o runtime Effect do SignatureKit. O escopo inicial é propositalmente estreito: **assinar PDFs com certificado A1 / PKCS#12 no navegador**. Todo o resto fica em pacotes de provider ou formato até o caminho A1 ficar boring.

```package-install
@signature-kit/react @signature-kit/a1 @signature-kit/pdf
```

## Registry install

```bash
npx shadcn@latest add a1-browser-signing --registry {URL_DA_DOCS}/registry
```

O item da registry (`a1-browser-signing`) envia apenas o bloco de UI:

```ts
// componentes copiados para sua app
components/a1-browser-signing.tsx
```

Ele depende dos pacotes da sua app (`@signature-kit/react`, `@signature-kit/a1`, `@signature-kit/pdf`); o item da registry adiciona só dependências locais de UI/form/store e nenhuma superfície de providers remotos de assinatura.

## Forma pública

| Import | Uso |
| --- | --- |
| `@signature-kit/react/browser-pdf` | Ler um `File` do navegador, derivar a geometria das páginas, montar o posicionamento da assinatura A1 e assinar um ou mais PDFs com PAdES. |
| `@signature-kit/react/components` | UI headless de posicionamento (`SignatureBuilderSurface`) mais store/selectors para a demo A1. Escritas de template retornam `Effect`. |
| `@signature-kit/react/builder` | Funções Effect puras para o template de assinatura A1: validar, posicionar/mover/remover um campo `signature` e derivar o widget do PDF. |
| `@signature-kit/react/config` | Schemas, tipos derivados e `ReactIntegrationError`. |

## Regra de runtime

React não é dono da assinatura. O navegador lê arquivos e recebe eventos; Effect possui o workflow; o signer A1 é fornecido na borda da aplicação:

```ts title="sign.ts"
import { a1SignaturesLayer } from "@signature-kit/a1/signer"
import { signBrowserPdf } from "@signature-kit/react/browser-pdf"
import { Effect, Redacted } from "effect"

const signed = await Effect.runPromise(
  signBrowserPdf({ pdf, template, fieldId: "sig" }).pipe(
    Effect.provide(a1SignaturesLayer({ pfx, password: Redacted.make(password) })),
  ),
)
```

Sem `runPromise` escondido, sem `Effect.provide` escondido, sem UX genérica de signer dentro do pacote React.
