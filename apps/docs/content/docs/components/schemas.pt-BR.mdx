Tudo o que entra ou sai do builder é descrito por um `Schema` do Effect. Eles vivem em `@signature-kit/react/config` e são a mesma fonte da verdade usada para decodificar entradas, derivar os tipos `*Input`/`*` e validar o que você passa para a [Builder API](/docs/components/builder-api). Cada `Schema` carrega seu tipo no campo `["Type"]`, então o tipo derivado e o validador nunca ficam fora de sincronia.

```ts title="config.ts"
import {
  ReactSignatureTemplateSchema,
  type ReactSignatureTemplate,
} from "@signature-kit/react/config"

// The type is the schema's own ["Type"] — there is no parallel manual definition.
export type Template = (typeof ReactSignatureTemplateSchema)["Type"]
```

## ReactIntegrationError

Toda fronteira do pacote falha com um único erro tagueado: `ReactIntegrationError`. Ele é uma `Schema.TaggedErrorClass`, então aparece como o canal `E` de todo `Effect` (`Effect.Effect<A, ReactIntegrationError>`) e é discriminável pela tag `"ReactIntegrationError"`.

<TypeTable
  type={{
    code: { type: "ReactIntegrationErrorCode", required: true, description: 'Stable, narrow code, e.g. "react.EMPTY_TEMPLATE" or "react.FIELD_OUT_OF_BOUNDS". Use ReactIntegrationErrorCodeValue.' },
    retryable: { type: "boolean", required: true, description: "Whether the operation can be retried without changing the input (e.g. I/O vs. validation failures)." },
    reason: { type: "string", description: "Human-readable message. It is also the message getter (falls back to code when absent)." },
    operation: { type: "ReactIntegrationOperation", description: 'Operation that failed, e.g. "react.builder.validate" or "react.browser.pdf.sign".' },
    schemaName: { type: "ReactIntegrationSchemaName", description: 'Schema that rejected the input, e.g. "ReactSignatureTemplate". Present on decode failures.' },
    issuePath: { type: "string", description: "Path of the field that failed Schema decoding (when present)." },
    issueMessage: { type: "string", description: "Effect Schema issue message for that path (when present)." },
  }}
/>

### Códigos (`ReactIntegrationErrorCode`)

`react.INVALID_BUILDER_INPUT` · `react.EMPTY_TEMPLATE` · `react.DUPLICATE_ID` · `react.UNKNOWN_DOCUMENT` · `react.UNKNOWN_ROLE` · `react.UNKNOWN_FIELD` · `react.FIELD_OUT_OF_BOUNDS` · `react.NO_AVAILABLE_PLACEMENT` · `react.INVALID_SIGNATURE_IMAGE` · `react.FILE_READ_FAILED` · `react.PDF_LOAD_FAILED` · `react.RENDER_FAILED` · `react.QUEUE_REJECTED`

As constantes nomeadas estão em `ReactIntegrationErrorCodeValue` (por exemplo, `ReactIntegrationErrorCodeValue.emptyTemplate`), as operações em `ReactIntegrationOperationValue` e os nomes de schema em `ReactIntegrationSchemaNameValue` — sempre prefira essas constantes a literais soltos.

### Como o erro chega ao canal tipado

Cada função do builder valida a entrada com `Schema.decodeUnknownEffect(...)` e, dentro de `Effect.mapError`, converte o `ParseError` do Effect em um `ReactIntegrationError` — preenchendo `code`, `operation`, `schemaName` e o par `issuePath` / `issueMessage` extraído da issue. As falhas de regra (template vazio, id duplicado, campo fora dos limites) usam `Effect.fail(new ReactIntegrationError(...))` diretamente. Em ambos os casos o erro permanece no canal `E` — ele nunca vira uma exceção solta.

```ts title="errors.tsx"
import { Effect } from "effect"
import { ReactIntegrationErrorCodeValue } from "@signature-kit/react/config"

// The E channel is already ReactIntegrationError — discriminate by code at the app boundary.
Effect.runPromise(store.placeField(placement)).catch((error) => {
  if (error.code === ReactIntegrationErrorCodeValue.fieldOutOfBounds) {
    toast("The field went off the page — reposition it.")
    return
  }
  if (error.retryable) {
    void retry()
    return
  }
  toast(error.reason ?? error.code)
})
```

<Callout type="info" title="Discrimine, não faça parse de string">
  Sempre ramifique por `error.code` (estável) e use `error.retryable` para decidir se vale repetir. `reason` / `issueMessage`
  servem para exibir ao usuário ou para logar, não para lógica.
</Callout>
