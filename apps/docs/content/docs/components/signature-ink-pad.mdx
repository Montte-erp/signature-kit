`SignatureInkPad` is the canvas where someone draws a signature by hand (mouse, pen, or touch). It captures the strokes via Pointer Events, redraws at high resolution using the `devicePixelRatio` and, at the end of each stroke, emits the image as a PNG data URL through `onChangeDataUrl`. It is a purely visual component: it never touches a key, never runs `Effect`, and depends on no secret.

```package-install
@signature-kit/react
```

## Anatomy

The pad is composed of three parts — the root, the `<canvas>`, and the clear button — and each one exposes a `data-slot`, so you can style it by class (`canvasClassName`, `clearButtonClassName`) or by inline style without changing the drawing logic.

```tsx title="DrawSignature.tsx"
import { SignatureInkPad } from "@signature-kit/react/components"
import { useState } from "react"

function DrawSignature() {
  const [dataUrl, setDataUrl] = useState<string | null>(null)

  return (
    <SignatureInkPad
      width={420}
      height={160}
      onChangeDataUrl={setDataUrl}
      onClear={() => setDataUrl(null)}
    />
  )
}
```

| Slot | Element | Description |
| ---- | ------- | ----------- |
| `signature-ink-pad` | div | Root of the pad; stacks the canvas and the clear button (`display: grid`). |
| `signature-ink-pad-canvas` | canvas | The drawing surface. Receives the Pointer Events and generates the PNG via `toDataURL`. |
| `signature-ink-pad-clear` | button | Button that resets the canvas to the `backgroundColor` and fires `onClear`. |

<Callout type="info" title="Data URL validated by schema — no secret">
  `onChangeDataUrl` delivers a PNG data URL (`canvas.toDataURL("image/png")`). This is exactly the format that the builder
  accepts in `imageDataUrl`, validated by the schema `^data:image\/(png|jpeg|webp);base64,[A-Za-z0-9+/=]+$`. The pad does not sign
  anything and does not depend on any key or secret: it only captures the drawing. The cryptographic signature stays at the
  `Signatures` boundary — here you only collect the _image_ of the handwritten signature.
</Callout>

## Behavior notes

- The canvas is resized by the `devicePixelRatio` on mount, so the signature stays sharp on retina screens without distorting the logical coordinates (`width`/`height`).
- `touchAction: "none"` on the canvas prevents scrolling while drawing by touch — the gesture becomes a stroke, not a scroll.
- `onChangeDataUrl` fires **on pointer release** (end of each stroke), not on every `pointermove`; use the last received value as the current image.
- Clearing reinitializes the canvas to `backgroundColor` and fires `onClear` (but not `onChangeDataUrl`).
