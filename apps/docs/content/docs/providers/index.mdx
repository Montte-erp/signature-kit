## No gateway

SignatureKit has no provider-normalizing package. Each remote signer lives in `signers/*`, exposes its own `create*SignatureRequest(...)` function, and shares the `SignatureHttpClient` from `@signature-kit/core/http`. Provider-specific options stay in their own package; there's no central dispatcher.

```package-install
@signature-kit/core @signature-kit/clicksign @signature-kit/assinafy @signature-kit/zapsign @signature-kit/docuseal @signature-kit/documenso
```

<Callout type="error" title="Redacted secrets">
  Wrap every token and API key in `Redacted.make(...)`. This keeps credentials out of logs, error messages, and
  stack traces. Never pass a secret in plain text or log it outside `Redacted`.
</Callout>

## Same input shape, provider-owned protocol

All remote packages consume a `RemoteSignatureRequestInput`: `title`, `documents`, `recipients`, optional `subject`, `message`, `send`, `expiresAt`, and `redirectUrl`. The provider package owns the protocol-specific translation.

```ts title="remote-provider.ts"
import { createDocumensoSignatureRequest } from "@signature-kit/documenso"
import { signatureHttpClientLive } from "@signature-kit/core/http"
import { Effect, Redacted } from "effect"

const request = yield* createDocumensoSignatureRequest(
  {
    apiKey: Redacted.make(process.env.DOCUMENSO_API_KEY ?? ""),
  },
  {
    title: "Service agreement",
    documents: [{ fileName: "contract.pdf", mimeType: "application/pdf", content }],
    recipients: [{ name: "Ana Souza", email: "ana@example.com", role: "signer" }],
    send: true,
  },
).pipe(Effect.provide(signatureHttpClientLive))
```

## Provider matrix

| Provider | Package | Primary API call | Draft support |
| --- | --- | --- | --- |
| Clicksign | `@signature-kit/clicksign` | v1 document + signer + notify flow | yes |
| Assinafy | `@signature-kit/assinafy` | account document + signer + assignment flow | yes |
| ZapSign | `@signature-kit/zapsign` | `POST /docs` | yes |
| DocuSeal | `@signature-kit/docuseal` | `POST /submissions/pdf` | yes |
| Documenso | `@signature-kit/documenso` | `POST /envelope/create` + optional `POST /envelope/distribute` | yes |

## Typed errors

All functions return `Effect.Effect<RemoteSignatureRequest, SignatureKitError, SignatureHttpClient>`. Input, HTTP, unsupported operation, and response-shape failures stay in Effect's typed channel with `provider`, `operation`, `schemaName`, and `status` whenever that data exists.
