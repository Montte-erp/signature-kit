## Sem gateway

O SignatureKit não tem um pacote que normalize provedores. Cada signatário remoto vive em `signers/*`, expõe sua própria função `create*SignatureRequest(...)` e compartilha o `SignatureHttpClient` de `@signature-kit/core/http`. As opções de cada provedor ficam no próprio pacote; não há despachante central.

```package-install
@signature-kit/core @signature-kit/clicksign @signature-kit/assinafy @signature-kit/zapsign @signature-kit/docuseal @signature-kit/documenso
```

<Callout type="error" title="Segredos protegidos com Redacted">
  Envolva todo token e API key em `Redacted.make(...)`. Isso mantém credenciais fora de logs, mensagens de erro e
  stack traces. Nunca passe um segredo em texto puro nem o registre em log fora de `Redacted`.
</Callout>

## Mesmo formato de entrada, protocolo de responsabilidade do provedor

Todos os pacotes remotos consomem um `RemoteSignatureRequestInput`: `title`, `documents`, `recipients`, e opcionalmente `subject`, `message`, `send`, `expiresAt` e `redirectUrl`. O pacote do provedor faz a tradução específica do protocolo.

```ts title="remote-provider.ts"
import { createDocumensoSignatureRequest } from "@signature-kit/documenso"
import { signatureHttpClientLive } from "@signature-kit/core/http"
import { Effect, Redacted } from "effect"

const request = yield* createDocumensoSignatureRequest(
  {
    apiKey: Redacted.make(process.env.DOCUMENSO_API_KEY ?? ""),
  },
  {
    title: "Service agreement",
    documents: [{ fileName: "contract.pdf", mimeType: "application/pdf", content }],
    recipients: [{ name: "Ana Souza", email: "ana@example.com", role: "signer" }],
    send: true,
  },
).pipe(Effect.provide(signatureHttpClientLive))
```

## Matriz de provedores

| Provedor | Pacote | Chamada de API principal | Suporte a rascunho |
| --- | --- | --- | --- |
| Clicksign | `@signature-kit/clicksign` | fluxo v1 de documento + signatário + notificação | sim |
| Assinafy | `@signature-kit/assinafy` | fluxo de documento + signatário + atribuição na conta | sim |
| ZapSign | `@signature-kit/zapsign` | `POST /docs` | sim |
| DocuSeal | `@signature-kit/docuseal` | `POST /submissions/pdf` | sim |
| Documenso | `@signature-kit/documenso` | `POST /envelope/create` + `POST /envelope/distribute` opcional | sim |

## Erros tipados

Todas as funções retornam `Effect.Effect<RemoteSignatureRequest, SignatureKitError, SignatureHttpClient>`. Falhas de entrada, de HTTP, de operação não suportada e de formato de resposta permanecem no canal tipado do Effect, com `provider`, `operation`, `schemaName` e `status` sempre que esses dados existirem.
