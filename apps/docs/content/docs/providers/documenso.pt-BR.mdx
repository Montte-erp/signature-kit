O Documenso é o adapter de assinatura remota open-source. O SignatureKit cria um envelope v2 com arquivos PDF e depois o distribui quando `send !== false`.

## Instalação

```package-install
@signature-kit/core @signature-kit/documenso
```

## Crie e distribua um envelope

Os exemplos públicos do Documenso usam o token de API diretamente em `Authorization`. Esse é o padrão do SignatureKit. Se a sua instância espera `Bearer <token>`, defina `authorizationScheme: "bearer"`.

```ts title="documenso.ts"
import { createDocumensoSignatureRequest } from "@signature-kit/documenso"
import { signatureHttpClientLive } from "@signature-kit/core/http"
import { Effect, Redacted } from "effect"

const request = yield* createDocumensoSignatureRequest(
  {
    apiKey: Redacted.make(process.env.DOCUMENSO_API_KEY ?? ""),
    baseUrl: "https://app.documenso.com/api/v2",
  },
  {
    title: "Service agreement",
    subject: "Please sign",
    message: "Review and sign this agreement.",
    documents: [{ fileName: "agreement.pdf", mimeType: "application/pdf", content: pdfBytes }],
    recipients: [{ name: "Ana Silva", email: "ana@example.com", role: "approver", routingOrder: 1 }],
    redirectUrl: "https://app.example.com/done",
  },
).pipe(Effect.provide(signatureHttpClientLive))

// request.provider === "documenso"
// request.state    === "sent" | "draft"
```

## Mapeamento da requisição

- `POST /envelope/create` recebe o `payload` multipart mais uma parte `files` por documento.
- Os destinatários são mapeados para papéis do Documenso: `approver` vira `APPROVER`; o padrão vira `SIGNER`.
- `routingOrder` é mapeado para `signingOrder`.
- `subject`, `message` e `redirectUrl` são mapeados para o `meta` do Documenso.
- Quando `send` é omitido ou true, o SignatureKit chama `POST /envelope/distribute` e retorna a primeira `signingUrl`.
- Quando `send: false`, o SignatureKit para após a criação e retorna uma requisição em rascunho.

<Callout type="info" title="Formato de expiração">
  O Documenso modela a expiração como um período de validade do envelope. O adapter atual não mapeia `expiresAt`; mantenha a política de expiração no próprio Documenso ou estenda as opções do provider quando precisar de controle exato.
</Callout>

## Alchemy

```ts title="alchemy.ts"
import * as Documenso from "@signature-kit/documenso"
import { signatureHttpClientLive } from "@signature-kit/core/http"
import { Layer, Redacted } from "effect"

export const providers = Documenso.providers({
  apiKey: Redacted.make(process.env.DOCUMENSO_API_KEY ?? ""),
}).pipe(Layer.provide(signatureHttpClientLive))
```

## Erros

Entradas inválidas, falhas HTTP remotas e falhas no formato da resposta resultam todas em `SignatureKitError`, com `provider: "documenso"` e metadados estruturados.
