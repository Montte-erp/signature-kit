title: Documenso
description: "Create Documenso v2 envelopes with multipart PDF uploads and typed SignatureKit errors."
---

Documenso cria um envelope v2, envia PDFs como multipart e pode distribuir o envelope quando `send` está ativo.

## Instalação

```package-install
@signature-kit/core @signature-kit/documenso
```

## Quando usar

Use Documenso quando posse do envelope, roteamento e ações de destinatário ficam no Documenso. Use assinatura local para bytes PDF/PAdES no processo.

<Callout type="error" title="Mantenha credenciais Redacted">
  Tokens de provider entram como `Redacted.make(...)` e são abertos apenas dentro da fronteira HTTP do pacote. Nunca passe tokens remotos para componentes React ou logs.
</Callout>

## Request mínimo

```ts title="documenso.ts"
import { signatureHttpClientLive } from "@signature-kit/core/http"
import { createDocumensoSignatureRequest } from "@signature-kit/documenso"
import { Effect, Redacted } from "effect"

const program = createDocumensoSignatureRequest(
  {
    apiKey: Redacted.make(process.env.DOCUMENSO_API_KEY ?? ""),
    baseUrl: "https://app.documenso.com/api/v2",
  },
  {
    title: "Contrato de serviço",
    message: "Revise e assine este documento.",
    documents: [{ fileName: "contract.pdf", mimeType: "application/pdf", content: pdfBytes }],
    recipients: [{ name: "Ana Silva", email: "ada@example.com", role: "signer" }],
    send: true,
  },
).pipe(Effect.provide(signatureHttpClientLive))

const request = await Effect.runPromise(program)
```


## Fatos do provider

- Usa a API v2 de envelope com upload multipart.
- `send: false` para após a criação e retorna um request em draft.
- Roles e ordem de assinatura mapeiam para campos do envelope Documenso.

## Erros

Input inválido, falhas HTTP remotas e shapes de resposta inválidos falham como `SignatureKitError`. Os campos `provider`, `operation` e `status` HTTP são preservados quando a resposta upstream expõe esses dados.
