DocuSign is a remote signer that runs directly on top of `SignatureHttpClient`: you pass the credentials and the normalized request, and the package assembles the envelope payload and the HTTP call. There is no intermediate gateway.

## Install

DocuSign is a direct remote signer. There is no intermediate gateway.

```package-install
@signature-kit/core @signature-kit/docusign
```

## Create an envelope

Pass the DocuSign credentials and the normalized request. The package assembles the envelope payload and calls `/v2.1/accounts/{accountId}/envelopes`.

```ts title="docusign.ts"
import { createDocuSignSignatureRequest } from "@signature-kit/docusign"
import { signatureHttpClientLive } from "@signature-kit/core/http"
import { Effect, Redacted } from "effect"

const request = yield* createDocuSignSignatureRequest(
  {
    baseUrl: "https://demo.docusign.net/restapi",
    accountId: process.env.DOCUSIGN_ACCOUNT_ID!,
    accessToken: Redacted.make(process.env.DOCUSIGN_TOKEN ?? ""),
  },
  {
    title: "Service agreement",
    subject: "Signature for contract 2026/0042",
    message: "Please review and sign the attached document.",
    documents: [{ fileName: "contract.pdf", mimeType: "application/pdf", content: pdfBytes }],
    recipients: [{ name: "Ana Souza", email: "ana@example.com", role: "signer" }],
    send: true,
  },
).pipe(Effect.provide(signatureHttpClientLive))

// request.provider === "docusign"
// request.state    === "sent" | "draft"
```

<Callout type="error" title="Credentials">
  Never hardcode the `accessToken` or the `accountId` in your code. Load them from environment variables and wrap the token in
  `Redacted.make(...)` — it stays redacted up to the HTTP boundary and never leaks into logs.
</Callout>

## Alchemy

For Alchemy stacks, the package exports `providers(options)` as a providers layer and keeps `SignatureHttpClient` explicit.

```ts title="alchemy.ts"
import * as DocuSign from "@signature-kit/docusign"
import { signatureHttpClientLive } from "@signature-kit/core/http"
import { Layer, Redacted } from "effect"

export const providers = DocuSign.providers({
  baseUrl: "https://demo.docusign.net/restapi",
  accountId: process.env.DOCUSIGN_ACCOUNT_ID!,
  accessToken: Redacted.make(process.env.DOCUSIGN_TOKEN ?? ""),
}).pipe(Layer.provide(signatureHttpClientLive))
```

## Errors

Invalid input, remote HTTP, and response shape all fail as `SignatureKitError`, with `provider: "docusign"` and structured metadata.

<Callout type="info" title="Typed error channel">
  Every failure arrives through Effect's typed channel — never as an exception. Handle it with `Effect.catchTag` or `Effect.catchAll` and
  inspect the literal code to react by error type.
</Callout>
