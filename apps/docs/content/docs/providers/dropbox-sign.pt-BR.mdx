O Dropbox Sign é um adapter de signatário remoto sobre o endpoint v3 `signature_request/send`. O SignatureKit monta a requisição multipart diretamente e mantém a API key em `Redacted` até a camada HTTP.

## Install

```package-install
@signature-kit/core @signature-kit/dropbox-sign
```

## Enviar uma requisição de assinatura

```ts title="dropbox-sign.ts"
import { createDropboxSignSignatureRequest } from "@signature-kit/dropbox-sign"
import { signatureHttpClientLive } from "@signature-kit/core/http"
import { Effect, Redacted } from "effect"

const request = yield* createDropboxSignSignatureRequest(
  {
    apiKey: Redacted.make(process.env.DROPBOX_SIGN_API_KEY ?? ""),
    testMode: true,
  },
  {
    title: "Mutual NDA",
    subject: "Please sign the NDA",
    message: "Review and sign when ready.",
    documents: [{ fileName: "nda.pdf", mimeType: "application/pdf", content: pdfBytes }],
    recipients: [{ name: "Jill Example", email: "jill@example.com", routingOrder: 1 }],
    redirectUrl: "https://app.example.com/after-sign",
    expiresAt: new Date("2030-01-01T00:00:00.000Z"),
  },
).pipe(Effect.provide(signatureHttpClientLive))

// request.provider === "dropbox-sign"
// request.state    === "sent"
```

## Mapeamento da requisição

- `documents[]` viram partes multipart `files[index]`.
- `recipients[]` viram `signers[index][name]`, `signers[index][email_address]` e, opcionalmente, `order`.
- `subject`, `message`, `clientId`, `redirectUrl` e `expiresAt` são mapeados para os campos de formulário do Dropbox Sign.
- `testMode: true` envia `test_mode=1`.

<Callout type="warn" title="Sem endpoint de rascunho">
  O `signature_request/send` do Dropbox Sign envia imediatamente. Passar `send: false` falha no canal tipado com
  `signature-kit.UNSUPPORTED_OPERATION`.
</Callout>

## Alchemy

```ts title="alchemy.ts"
import * as DropboxSign from "@signature-kit/dropbox-sign"
import { signatureHttpClientLive } from "@signature-kit/core/http"
import { Layer, Redacted } from "effect"

export const providers = DropboxSign.providers({
  apiKey: Redacted.make(process.env.DROPBOX_SIGN_API_KEY ?? ""),
  testMode: true,
}).pipe(Layer.provide(signatureHttpClientLive))
```

## Errors

Entrada inválida, requisições de rascunho não suportadas, falhas de HTTP remoto e de formato de resposta falham como `SignatureKitError`, com `provider: "dropbox-sign"`.
